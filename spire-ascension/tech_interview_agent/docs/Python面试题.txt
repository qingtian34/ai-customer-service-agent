Python中列表和元组的区别？
列表可变，支持增删改，作为字典键不可用。元组不可变，创建后不能修改，可作为字典键，占用内存略小。元组适合固定数据如坐标、配置项；列表适合动态集合。

什么是装饰器？如何实现？
装饰器是接受函数并返回新函数的高阶函数，用于在不修改原函数代码的情况下扩展功能。常用@语法糖。典型应用：日志、权限校验、计时、缓存（functools.lru_cache）。

生成器和迭代器的区别？
迭代器实现__iter__和__next__，生成器是特殊的迭代器，用yield创建，惰性求值节省内存。生成器适合处理大数据流或无限序列，如读取大文件逐行处理。

GIL是什么？对多线程有什么影响？
GIL（全局解释器锁）保证同一时刻只有一个线程执行Python字节码。CPU密集型多线程无法利用多核，应使用multiprocessing。IO密集型多线程仍有效，因IO等待时会释放GIL。

*args和**kwargs的作用？
*args接收任意数量的位置参数，打包为元组。**kwargs接收任意数量的关键字参数，打包为字典。常用于函数包装和转发参数，如装饰器内部调用原函数。

深拷贝和浅拷贝的区别？
浅拷贝只复制对象本身，嵌套对象仍共享引用（copy.copy、切片）。深拷贝递归复制所有嵌套对象（copy.deepcopy）。修改浅拷贝的嵌套对象会影响原对象。

with语句的原理？
with基于上下文管理器，进入时调用__enter__，离开时调用__exit__处理清理。即使发生异常也会执行__exit__，常用于文件、锁、数据库连接的安全释放。

什么是列表推导式和字典推导式？
简洁创建列表/字典的语法：[x*2 for x in range(10) if x%2==0]。字典推导式：{k:v for k,v in items}。比循环更简洁，但复杂逻辑应保持可读性。

async/await如何工作？
async def定义协程，await挂起当前协程让出控制权。asyncio事件循环调度多个协程并发执行IO操作。适合高并发网络服务，不适用于CPU密集型任务。

Python如何管理内存？
主要依赖引用计数，对象引用为0时立即回收。循环引用由分代垃圾回收器处理。小整数和短字符串有缓存机制。大对象使用内存池减少malloc调用。

is和==的区别？
==比较值是否相等，is比较是否为同一对象（id相同）。判断None应使用is None，因为None是单例。比较数值内容用==，不要用is。

@property的作用？
将方法装饰为属性访问，可配合setter实现校验。对外像访问属性，内部可封装逻辑。常用于数据类中计算属性或只读字段。

typing模块常用类型有哪些？
List、Dict、Optional、Union、Tuple、Callable等。Python 3.9+可用内置list、dict。类型注解提高可读性，mypy等工具可做静态检查，运行时不强制。

如何排查Python内存泄漏？
使用gc模块查看不可达对象，tracemalloc追踪分配，memory_profiler分析逐行内存。常见原因：全局列表持续增长、未关闭资源、循环引用持有大对象。

lambda表达式适用场景？
匿名短函数，常用于map、filter、sorted的key参数。复杂逻辑应写def函数。lambda只能包含表达式，不能写多条语句。

enumerate和zip的用法？
enumerate(iterable)同时返回索引和元素。zip(a,b)将多个可迭代对象按位置配对。zip最长长度以最短序列为准，可用zip_longest填充。

魔术方法__str__和__repr__的区别？
__repr__面向开发者，应尽可能唯一标识对象，理想情况eval(repr(obj))==obj。__str__面向用户，可读性优先。未定义__str__时print使用__repr__。

虚拟环境venv的作用？
隔离项目依赖，避免不同项目包版本冲突。python -m venv .venv创建，激活后pip install仅影响当前环境。生产部署应固定requirements.txt版本。

Python多进程如何通信？
Queue、Pipe用于进程间传递数据；multiprocessing.shared_memory共享内存；Manager提供跨进程对象。注意进程间不共享普通全局变量，需显式通信机制。
